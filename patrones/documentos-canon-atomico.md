# Documentos Canon Atomico

Patron para separar el atlas documental bruto del canon reusable cuando una superficie Windows o Codex acumula cronologia, snapshots, evidencia y materiales repetidos.

## Senales

- Hay una carpeta con mucha cronologia y pocas reglas visibles.
- Aparecen snapshots o inventarios mezclados con documentos vivos.
- El mismo tipo de evidencia reaparece en distintas waves.
- La ruta visible no aclara si algo es canon, copia o archivo.

## Regla

1. Tratar el inventario como atlas, no como canon.
2. Separar cronologia, snapshot, evidencia y reusable.
3. Promover a canon solo lo que repite o sobrevive una wave validada.
4. Mantener los alias visibles sin confundirlos con la fuente.
5. Registrar el cambio con readback corto y un solo siguiente delta.

## Clasificacion

- `atlas`: lista de lo que existe.
- `canon`: lo que se repite y merece quedarse reutilizable.
- `cronologia`: orden temporal, no decision.
- `snapshot`: lectura fija para control o comparacion.
- `evidencia`: prueba de un caso o una wave.

## Salida

Un atlas legible, un canon reusable y una superficie que no mezcla historia con regla.

## Stop Condition

- Falta evidencia repetida.
- Atlas y canon siguen mezclados.
- La ruta visible confunde snapshot con superficie activa.
- No hay readback de cierre.
